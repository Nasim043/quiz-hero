1. What is the difference between map, filter, forEach and find?

map():

  • This method create new array by calling the provided function in every element.
  • The map() method does not execute the function for array elements without values.
  • This method does not change the original array.

filter():

  • This method create new array with only elements passed condition inside the provided function.
  • filter() does not execute the function for array elements without values.
  • filter() does not change the original array.

forEach():

  • This method loop over array’s items and perform an action on each item.
  • The forEach() method does not return an array.

find():

  • This method is used to search for a specific value in an array.
  • It returns the first matched element if found otherwise retrun undefined.

2. What is the difference between cookies, local storage and session storage?

cookies:

  • Used to store small amounts of data, for example, data for user identification.
  • expiration date can be set.
  • Can be shared between tabs.

local storage:

  • Shared between all tabs and windows with the same origin.
  • Survives browser restart.
  • It never expires.

session storage:

  • Visible within a browser tab, including iframes from the same origin.
  • Survives page refresh but not tab close.
  • Expires on the tab close.

3. What is event loop and how it works?

The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue.
Whenever asynchronous events like setTimeOut or promises happen if there is any I/O involved, it then takes some time to accomplish these tasks. They simply put into the message queue or callback queue if their I/O was complete or if there was none. The event loop first completes the call stack tasks, after which it will search for any callback or message queue tasks to perform.
